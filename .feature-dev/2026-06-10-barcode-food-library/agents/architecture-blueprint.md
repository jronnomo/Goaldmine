```typescript
/** Subset of OFF v2 product response — fields requested via ?fields=... */
export type OffProduct = {
  product_name?: string;
  brands?: string;
  serving_size?: string;
  // Preferred: per-serving energy in kcal
  "energy-kcal_serving"?: number;
  // Fallback: per-serving energy in kJ
  "energy_serving"?: number;
  // Per-100g equivalents (fallback when serving data absent)
  "energy-kcal_100g"?: number;
  "energy_100g"?: number;
  nutriments?: {

    "salt_serving"?: number;

    fat_100g?: number;
    fiber_100g?: number;
    "sodium_100g"?: number;
```

**OFF v2 endpoint URL pattern** (fields param must list exactly these keys):
```
https://world.openfoodfacts.org/api/v2/product/{barcode}?fields=product_name,brands,serving_size,energy-kcal_serving,energy_serving,energy-kcal_100g,energy_100g,nutriments
```

### 4.2 `normalizeOffProduct` — Full Rules with Pseudocode

**Decision: pipe sanitization at normalizer level.** Strips `|` from `name` and `brand` on the way in, before writing to the DB. Reasoning:
1. `LibraryFood.name` and `.brand` carry a DB-level guarantee of pipe-free strings — all consumers (chip render, `onAdd` line builder, MCP response) are unconditionally safe.
2. The normalizer already performs lossy transforms (unit conversion, rounding, null discipline) — pipe-stripping is one more data-cleaning step of the same character.
3. Sanitizing at `onAdd` would require every current and future consumer to independently remember the hazard; a single ingest-time strip is the safer invariant.
4. Alternative: sanitize in `onAdd` only. Rejected because the DB would then hold raw OFF data that could silently corrupt on any new consumer that assembles a pipe-delimited string.

```typescript
export function normalizeOffProduct(raw: OffProduct, barcode: string): {
  name: string;
  brand: string | null;
  servingSize: string | null;
  basis: "serving" | "100g";
  macros: FoodMacros;
} {
  // ── Name ────────────────────────────────────────────────────────────────
  // Strip pipes; fall back to barcode if empty.
  const name = (raw.product_name?.trim().replace(/\|/g, "-") || barcode);

  // ── Brand ────────────────────────────────────────────────────────────────
  // OFF brands field is comma-separated list; take the first segment.
  // Strip pipes from the result.
  const brandRaw = raw.brands?.split(",")[0]?.trim() ?? null;
  const brand = brandRaw ? brandRaw.replace(/\|/g, "-") : null;

  // ── Basis selection ──────────────────────────────────────────────────────
  // "serving" basis requires at least one nutriment_serving field to be present
  // AND serving_size to be a non-empty string (proves serving label is defined).
  // "100g" basis is the fallback.
  const hasServingData =
    !!raw.serving_size?.trim() &&
    (
      raw["energy-kcal_serving"] != null ||
      raw["energy_serving"] != null ||
      raw.nutriments?.proteins_serving != null ||
      raw.nutriments?.carbohydrates_serving != null ||
      raw.nutriments?.fat_serving != null
    );

  const basis: "serving" | "100g" = hasServingData ? "serving" : "100g";
  const servingSize = hasServingData ? (raw.serving_size?.trim() ?? null) : "100 g";

  // ── Helpers ──────────────────────────────────────────────────────────────
  const suffix = basis === "serving" ? "_serving" : "_100g";

  /** Return value if finite and >= 0, else null. */
  function safeNum(v: number | undefined | null): number | null {
    if (v == null || !Number.isFinite(v) || v < 0) return null;
    return v;
  }

  // ── Calories ────────────────────────────────────────────────────────────
  // Priority 1: energy-kcal_{suffix} (direct kcal field)
  // Priority 2: energy_{suffix} in kJ ÷ 4.184
  // NEVER derive calories from macros × Atwater.
  // NEVER use bare "energy" key (kJ) as kcal.
  let calories: number | null = null;
  const kcalDirect = safeNum(
    basis === "serving" ? raw["energy-kcal_serving"] : raw["energy-kcal_100g"]
  );
  if (kcalDirect != null) {
    calories = Math.round(kcalDirect);
  } else {
    const kjVal = safeNum(
      basis === "serving" ? raw["energy_serving"] : raw["energy_100g"]
    );
    if (kjVal != null) {
      calories = Math.round(kjVal / 4.184);
    }
  }

  // ── Protein, Carbs, Fat, Fiber ───────────────────────────────────────────
  // Source from nutriments object; 1 decimal rounding.
  function gram1dp(v: number | undefined | null): number | null {
    const n = safeNum(v);
    if (n == null) return null;
    return Math.round(n * 10) / 10;
  }
  const n = raw.nutriments ?? {};
  const proteinG  = gram1dp(n[`proteins${suffix}`]);
  const carbsG    = gram1dp(n[`carbohydrates${suffix}`]);
  const fatG      = gram1dp(n[`fat${suffix}`]);
  const fiberG    = gram1dp(n[`fiber${suffix}`]);

  // ── Sodium ───────────────────────────────────────────────────────────────
  // OFF stores sodium in grams → multiply by 1000 for mg.
  // If sodium missing but salt present: salt_g × 400 (NaCl 39.3% sodium ≈ 400 mg/g).
  // Result: integer (Math.round).
  let sodiumMg: number | null = null;
  const sodiumRaw = safeNum(n[`sodium${suffix}`]);
  if (sodiumRaw != null) {
    sodiumMg = Math.round(sodiumRaw * 1000);
  } else {
    const saltRaw = safeNum(n[`salt${suffix}`]);
    if (saltRaw != null) {
      sodiumMg = Math.round(saltRaw * 400);
    }
  }

  return {
    name,
    brand,
    servingSize,
    basis,
    macros: { calories, proteinG, carbsG, fatG, fiberG, sodiumMg },
  };
}
```

### 5.1 `lookupBarcode(raw: string): Promise<BarcodeLookupResult>`

**Multi-form barcode match set** (UPC-A ↔ EAN-13 interop):

Given a raw barcode string from the scanner or manual input:
```
raw         = "49000028911"        (11-digit — after leading zero stripped)
padded13    = "0049000028911"      (zero-pad to 13)
stripped    = raw.replace(/^0+/, "")  (strip leading zeros)
```

Always query all three forms in a single `findFirst` with `OR`:
```typescript
const barcode = raw.trim();
// Validate format before any DB/network work
if (!/^\d{8,14}$/.test(barcode)) {
  return { status: "not_found" }; // invalid digits — no action call
}

// Compute all barcode forms
const padded13 = barcode.padStart(13, "0");
const stripped = barcode.replace(/^0+/, "") || barcode; // guard: all-zeros edge case

// Library lookup
const existing = await prisma.foodLibrary.findFirst({
  where: { barcode: { in: [barcode, padded13, stripped] } },
});
```

If found → bump usageCount/lastUsedAt + return `found, fromLibrary: true`:
```typescript
if (existing) {
  await prisma.foodLibrary.update({
    where: { id: existing.id },
    data: { usageCount: { increment: 1 }, lastUsedAt: new Date() },
  });
  return { status: "found", food: toLibraryFood(existing), fromLibrary: true };
}
```

OFF fetch (only on library miss):
```typescript
const OFF_UA = "Goaldmine/1.0 (github.com/jronnomo/goaldmine)";
const OFF_FIELDS = [
  "product_name", "brands", "serving_size",
  "energy-kcal_serving", "energy_serving",
  "energy-kcal_100g", "energy_100g",
  "nutriments",
].join(",");

async function fetchOff(code: string): Promise<BarcodeLookupResult> {
  const url = `https://world.openfoodfacts.org/api/v2/product/${code}?fields=${OFF_FIELDS}`;
  try {
    const res = await fetch(url, {
      headers: { "User-Agent": OFF_UA },
      signal: AbortSignal.timeout(6000),
    });
    if (!res.ok) return { status: "not_found" };
    const json = await res.json() as { status: number; product?: unknown };
    if (json.status !== 1 || !json.product) return { status: "not_found" };
    const normalized = normalizeOffProduct(json.product as OffProduct, code);
    // Upsert — barcode as the unique key; use padded-13 form for storage
    const upserted = await prisma.foodLibrary.upsert({
      where: { barcode: padded13 },
      create: {
        barcode: padded13,
        name: normalized.name,
        brand: normalized.brand,
        servingSize: normalized.servingSize,
        basis: normalized.basis,
        ...normalized.macros,
        usageCount: 1,
        lastUsedAt: new Date(),
      },
      update: {
        // Re-normalize on re-encounter (OFF data can improve)
        name: normalized.name,
        brand: normalized.brand,
        servingSize: normalized.servingSize,
        basis: normalized.basis,
        ...normalized.macros,
        usageCount: { increment: 1 },
        lastUsedAt: new Date(),
      },
    });
    revalidatePath("/nutrition");
    return { status: "found", food: toLibraryFood(upserted), fromLibrary: false };
  } catch {
    return { status: "error", message: "Network error — try again" };
  }
}
```

**12-digit zero-pad retry**: If OFF returns `status !== 1` for `barcode` AND `barcode.length === 12`, retry once with `"0" + barcode` (UPC-A → EAN-13):
```typescript
let result = await fetchOff(barcode);
if (result.status === "not_found" && barcode.length === 12) {
  result = await fetchOff("0" + barcode);
}
return result;
```

**Error discipline**: the outer `try/catch` around fetch returns `{status:"error"}` — never throws to the client. Validation errors (bad regex) return `{status:"not_found"}` silently. Nothing re-throws beyond the `fetchOff` boundary.

```typescript
async function startCamera() {
  setScannerState("starting");
  try {
    const stream = await navigator.mediaDevices.getUserMedia({
      video: { facingMode: { ideal: "environment" }, width: { ideal: 1280 }, height: { ideal: 720 } },
    });
    streamRef.current = stream;
    if (videoRef.current) {
      videoRef.current.srcObject = stream;
      await videoRef.current.play();
    }
    // Init decoder once (not per-frame)
    await initDecoder();
    setScannerState("scanning");
  } catch (err) {
    const name = (err as DOMException).name;
    if (name === "NotAllowedError" || name === "PermissionDeniedError") {
      setScannerState("denied");
    } else {
      setScannerState("no-camera");
    }
  }
}
```

### 6.4 Decoder Strategy — Initialization (once per camera session)

```typescript
type Decoder = (imageData: ImageData) => Promise<string | null>;
const decoderRef = useRef<Decoder | null>(null);

async function initDecoder() {
  // Native BarcodeDetector path: check for EAN-13 support
  // NOTE: native format names use underscore: "ean_13" (not "EAN-13")
  if (typeof BarcodeDetector !== "undefined") {
    try {
      const supported = await BarcodeDetector.getSupportedFormats();
      if (supported.includes("ean_13")) {
        const detector = new BarcodeDetector({
          formats: ["ean_13", "upc_a", "upc_e", "ean_8"],
        });
        decoderRef.current = async (img) => {
          const results = await detector.detect(img);
          return results[0]?.rawValue ?? null;
        };
        return; // native path established
      }
    } catch {
      // BarcodeDetector exists but getSupportedFormats failed — fall through to zxing
    }
  }

  // zxing-wasm path (iOS Safari, Firefox, and everywhere native is absent)
  // Dynamic import: separate lazy chunk, not in base bundle
  const { prepareZXingModule, readBarcodes } = await import("zxing-wasm/reader");

  // Warm the wasm with the public/ locateFile override
  await prepareZXingModule({
    overrides: {
      locateFile: (path: string, prefix: string) => {
        if (path.endsWith(".wasm")) return `/zxing/${path}`;
        return prefix + path;
      },
    },
    fireImmediately: true, // returns Promise<ZXingReaderModule>; awaitable
  });

  // PRD format strings (HRI labels) are valid per research v3.1.0 verification
  decoderRef.current = async (img) => {
    const results = await readBarcodes(img, {
      formats: ["EAN-13", "UPC-A", "UPC-E", "EAN-8"],
      tryHarder: true,
      maxNumberOfSymbols: 1,
    });
    const valid = results.find(r => r.isValid && r.text);
    return valid?.text ?? null;
  };
}
```

### 6.5 Decode Loop (runs while `scannerState === "scanning"`)

```typescript
useEffect(() => {
  if (scannerState !== "scanning" || !decoderRef.current) return;

  let cancelled = false;
  let inFlight = false;
  let lastCode: string | null = null;
  let matchCount = 0;

  const loop = async () => {
    while (!cancelled) {
      await new Promise<void>(r => setTimeout(r, 200));
      if (cancelled || inFlight) continue;

      const video = videoRef.current;
      if (!video || video.readyState < HTMLMediaElement.HAVE_CURRENT_DATA) continue;

      inFlight = true;
      try {
        // Capture frame onto ≤720px canvas (reduce decode cost; wasm faster on smaller images)
        const W = Math.min(video.videoWidth, 1280);
        const H = Math.round(W * (video.videoHeight / video.videoWidth));
        const canvas = canvasRef.current!;
        canvas.width = W;
        canvas.height = H;
        const ctx = canvas.getContext("2d")!;
        ctx.drawImage(video, 0, 0, W, H);
        const imageData = ctx.getImageData(0, 0, W, H);

        const code = await decoderRef.current!(imageData);
        if (!cancelled && code) {
          if (code === lastCode) {
            matchCount++;
            if (matchCount >= 2) {
              navigator.vibrate?.(50); // only feedback flourish
              onDetected(code);
              return; // stop loop — parent will change phase

          } else {
            lastCode = code;
            matchCount = 1;
          }
        }
      } catch {
        // decode errors are transient; continue loop
      } finally {
        inFlight = false;
      }
    }
  };

  loop();
  return () => { cancelled = true; };
}, [scannerState, onDetected]);
```

```typescript
  // name/brand are pipe-free (normalizer-level guarantee) — no replace needed here
  const brandPart = food.brand ? ` (${food.brand})` : "";
  const qty = food.basis === "100g"
    ? `${Math.round(servings * 100)} g`
    : `${servings} serving${servings === 1 ? "" : "s"}`;
  const line = `${food.name}${brandPart} | ${qty}`;
  setItemsText(prev => prev + (prev.trim() ? "\n" : "") + line);

  // 2. Sum macros (null keys untouched; null food values skip)
  setMacros(prev => {
    const next = { ...prev };
    for (const key of MACRO_KEYS) {
      const foodVal = food.perServing[key];
      if (foodVal == null) continue; // null: leave existing value alone
      const scaled = foodVal * servings;
      const existing = prev[key] ?? 0;
      const sum = existing + scaled;
      if (key === "calories" || key === "sodiumMg") {
        next[key] = Math.round(sum);

        // gram fields: 1 decimal
        next[key] = Math.round(sum * 10) / 10;

  });

}
```

### 10.4 Vercel Postinstall Ordering Note

Vercel runs the following for a Node.js project:
1. `npm ci` (installs all deps including `zxing-wasm@3.1.0`)

## 13. Critical Decisions with Reasoning

| # | Decision | Reasoning | Alternative Considered |
|---|----------|-----------|----------------------|
| 1 | **Pipe sanitization at normalizer level** | `LibraryFood.name/brand` carry a DB-wide pipe-free guarantee; all current and future consumers are unconditionally safe. Single ingest-time transform vs. remembering the hazard at every consumer. | Sanitize in `onAdd` only — rejected because new consumers would inherit the raw-pipe risk silently. |
| 2 | **`zxing-wasm` exact pin `"3.1.0"`** | API is researched against this exact version; minor version bumps could change `prepareZXingModule` semantics. Exact pin = zero surprise on `npm ci`. | `^3.1.0` — rejected because the research confirmed exact API compatibility, and wasm packages can have breaking minor bumps. |
| 3 | **Decoder initialized once per camera session** | `BarcodeDetector.getSupportedFormats()` is async and the formats don't change mid-session. `prepareZXingModule` warms the wasm — calling it once is the intent. Per-frame init would be a performance regression. | Per-frame decoder construction — rejected. |
| 4 | **Two consecutive identical reads acceptance** | Prevents spurious reads from partial scans, dirty barcodes, or motion blur. One-read acceptance would cause false positives. The 200ms loop with skip-if-in-flight means two confirmations take ≤600ms — well within perceived real-time. | Single read — rejected; three reads — too slow (>800ms) for a fast scan. |
| 5 | **Manual strip always rendered, no camera state blocks it** | US-004 ("never blocked"). The strip is in `ScanFoodSheet`, not inside `BarcodeScanner`. Camera states (denied, no-camera, error) only affect the viewfinder card, not the strip below it. | Hide strip during scanning — rejected; would break the fallback UX. |
| 6 | **`initialFood` prop bypasses scan phase entirely** | Chip taps should feel instant (no camera lifecycle). When `initialFood` is set, `BarcodeScanner` never mounts, no camera permission fires, no tracks to clean up. The scanner's complex lifecycle is zero-cost for the common chip case. | Always open scan phase, prefill manual field — rejected; unnecessary camera churn. |
| 7 | **`LogNutritionForm` dual-fetch strategy** | `/nutrition` page is a server component and can call `getQuickPickFoods()` at render time with no extra latency. `LogLauncher` is client-only; `LogNutritionForm` mounts lazily (accordion open) and fires a `useEffect` server action call. No root-layout DB threading required. | Fetch in `LogLauncher` and pass as prop — rejected; `LogLauncher` is client-side and would need a server component wrapper or RSC boundary. |
| 8 | **`frequentFoods` inside existing `safe()` call** | Atomic with the nutrition query; consistent error shape. Adding a separate top-level `safe()` call would mean partial success (byDay but no frequentFoods) surfacing as a confusing half-response. | Separate safe() for frequentFoods — rejected; partial results would confuse the coach. |
| 9 | **Upsert uses padded-13 form as the canonical barcode storage key** | EAN-13 is the universal standard; storing the padded form means the `@unique` index is always hit with the padded-13 even when the scanner returns UPC-A. The OR query in library lookup handles the reverse case (OFF might return 12-digit). | Store raw scanner code — rejected; would fragment the library across UPC-A and EAN-13 forms of the same barcode. |
| 10 | **No `recordFoodUse` call for scan paths** | `lookupBarcode` itself increments `usageCount` and sets `lastUsedAt` (both on library hits and new upserts). A second `recordFoodUse` call in `handleAdd` for scan paths would double-count. The `chipSource` flag in `AddFoodPayload` disambiguates the chip-only case. | Always call `recordFoodUse` in `handleAdd` — rejected; double-increment for scan paths. |

## 15. Risks

| # | Risk | Severity | Mitigation |
|---|------|----------|-----------|
| R1 | **iOS camera requires HTTPS** — `getUserMedia` fails on `http://localhost:3000` | High | `npm run dev:https` (`next dev --experimental-https`) is the dev camera path. Documented in package.json scripts. Deployment on Vercel is always HTTPS — no prod risk. Manual strip is always the fallback. |
| R2 | **Neon migration is shared with prod** — a bad `ALTER` on a live table could corrupt data | High | Blueprint specifies additive-only SQL review gate before `prisma migrate dev --confirm`. Expected SQL: CREATE TABLE + 2 indexes. Any DROP/ALTER = abort. |
| R3 | **wasm chunk size (1.04 MiB)** — long initial load if the user opens the scanner on a slow connection | Medium | Loaded lazily behind `next/dynamic` + `import("zxing-wasm/reader")` — base bundle is unaffected (PRD §8.3 gate). zxing-wasm is initialized inside `initDecoder()` which only fires on camera session start. On repeat opens, the wasm is cached by the browser. |
| R4 | **iOS visibilitychange race** — phone sleeps/wakes while `startCamera()` is still in-flight | Medium | The `active` flag + `streamRef` are checked before `stopCamera()` is called; the stream reference may be null if the camera didn't finish starting. The `startCamera()` catches any error from an interrupted `getUserMedia`. Manual strip covers any state. |
| R5 | **OFF API reliability** — OFF can be down, slow, or return malformed JSON | Medium | `AbortSignal.timeout(6000)` limits wait. `try/catch` returns `{status:"error"}`. User sees error state + retry + manual strip. No data written on error. |
| R6 | **BarcodeDetector format name collision** — native API uses `ean_13` (underscore); zxing uses `"EAN-13"` (hyphen) | Medium | Separate code paths in `initDecoder()` — native path uses native format strings, zxing path uses PRD's HRI strings. They never mix. Documented clearly in code comments. |
| R7 | **Connector cache** — after `tools.ts` changes (frequentFoods), claude.ai may serve stale tool schema | Low | `MCP_SERVER_VERSION` stamps off `VERCEL_GIT_COMMIT_SHA` (per project-gotchas.md §C). Every deploy bumps the version → connector re-fetches. Toggle connector off/on if needed. |
| R8 | **Double usage-count increment** — `lookupBarcode` already bumps count; `recordFoodUse` in `handleAdd` would double it for scan paths | Low | Resolved by `chipSource` flag in `AddFoodPayload`. Blueprint §8.2 specifies the conditional call. |
| R9 | **parseMacros reads named form fields** — if MacroInputs `name` attributes change, `logNutrition` silently drops macros | Low | Blueprint explicitly locks field names (`"calories"`, `"proteinG"`, etc.) and the controlled mode change preserves them. Regression gate: EditNutritionForm submission must still record correct macros. |
| R10 | **`next/dynamic ssr:false` in nested client component** — confirmed working per research §5; could regress in Next 16 minor bumps | Low | Confirmed against Next.js 16.2.4 source. Acceptance gate §8.3 verifies lazy chunks in build output. |
